Change log for opb_emc, 2.00.a


Changes in 8.1.1, Update # 050

8.1.1 - Changes in tool interface files (.mpd) 

Correct IO_IS typo of the following parameters:C_THZOE_PS_MEM_0, C_THZOE_PS_MEM_1, C_THZOE_PS_MEM_2 and C_THZOE_PS_MEM_3.


Changes in 8.1, Update # 040

8.1 - Changes in tool interface files (.mpd) 

Added individual _I, _O and _T ports for Tri-state port Mem_DQ.

8.1 - Changes in TCL files associated with core (.tcl) 


- clean up old datastructure APIs


Changes in 7.1.2, Update # 030

7.1.2 - Changes in VHDL sources (.vhd) 


Fixed issue with Mem_DQ_T registers not being packed into the IOB's


Changes in 7.1.1, Update # 020

7.1.1 - Changes in VHDL sources (.vhd) 


Fixed problem with ZERO_DATA constant being of the wrong width when
C_INCLUDE_DATAWIDTH_MATCHING_0 = 0 and the memory device width was
defined as something other than 32-bits wide.

7.1.1 - Changes in tool interface files (.mpd) 

Added full support for spartan3e.


Changes in 6.3.2, Update # 010

6.3.2 - Changes in VHDL sources (.vhd) 


This New version, opb_emc_v2_00_a, supports OPB sequential address
transactions with a burst response.

The following Parameters have been modified:
C_INCLUDE_BURST can now be set to 0 or 1. This version supports
sequential address transactions in a burst-like manner. Default value
is now 0.

Fixed problem with memory state machine missing single beat cycles while
waiting for read or write recovery.

6.3.2 - Changes in tool interface files (.mpd) 

Added in 6.3.2.

6.3.2 - Changes in TCL files associated with core (.tcl) 


Added tcl for 2.00.a version
